# 말씀암송 인증방 PWA — UI 설계 기준

> 프론트엔드 UI/UX 작업 시 이 문서를 기준으로 한다.  
> DB 구조는 [`DB_SPEC.md`](./DB_SPEC.md)를 따른다.

---

## 디자인 방향

| 원칙 | 설명 |
|------|------|
| **모바일 PWA 우선** | 스마트폰 세로 화면 기준. 데스크톱은 보조. |
| **따뜻하고 단순** | 소규모 교회·모임에 어울리는 부드러운 톤. 과한 장식·기업형 UI 지양. |
| **관리자 시스템처럼 보이지 않게** | CRUD 중심이 아닌, 매일 함께하는 암송 공간 느낌. |
| **가벼운 피드형** | SNS처럼 무겁지 않고, 매일 인증하고 싶게 만드는 구조. |
| **상태가 명확** | 오늘 인증 완료/미완료가 홈·방 상세에서 한눈에 보여야 함. |

### 톤 & 무드

- 색: 따뜻한 중성 톤 + 포인트 컬러 1~2개 (예: 부드러운 베이지·크림 + 차분한 블루/그린)
- 타이포: 가독성 높은 sans-serif, 본문(말씀)은 여백 충분
- 아이콘: 이모지 또는 단순 라인 아이콘 (🎙️ 녹음, 🔢 계수기, 🙏 아멘)
- 여백: 카드 간 충분한 간격, 터치 영역 44px 이상

## 피드 기본 구조 (핵심)

암송방 상세의 인증 피드는 **반드시** 아래 계층을 따른다.

```
날짜 구분선          ← CHECK_IN_DATE
  └─ 사람별 인증 카드   ← CHECK_IN_ID (+ MEMBER 정보)
       └─ 녹음/계수기 기록  ← DETAIL_ID 목록
       └─ 아멘 버튼 + 아멘 수  ← TB_RECITATION_AMEN COUNT
```

### 시각 예시

```
─── 2026년 6월 13일 ───

┌─────────────────────────┐
│ 👤 김윤기                │
│ 🎙️ 녹음 인증    09:12   │
│ 🔢 계수기 10회  09:30   │
│ 🙏 아멘 3    [아멘]     │
└─────────────────────────┘

┌─────────────────────────┐
│ 👤 박은빈                │
│ 🎙️ 녹음 인증    10:05   │
│ 🙏 아멘 2    [아멘]     │
└─────────────────────────┘
```

### DB ↔ UI 매핑

| UI 요소 | DB 기준 |
|---------|---------|
| 날짜 구분선 | `CHECK_IN_DATE` |
| 사람별 카드 | `CHECK_IN_ID` |
| 카드 내 기록 | `DETAIL_ID` (`STATUS='Y'`) |
| 아멘 수 | `TB_RECITATION_AMEN` COUNT (`STATUS='Y'`) |
| 내 아멘 여부 | 현재 MEMBER의 AMEN row `STATUS='Y'` |
| 인증 완료 표시 | `TB_RECITATION_CHECK_IN.STATUS` |

## 화면별 UI 규칙

### 1. 랜딩페이지

- 서비스 핵심 메시지 노출: *「함께 말씀을 암송하고, 매일 인증하며, 서로 아멘으로 격려하는 암송방」*
- CTA 1개: 「카카오로 시작하기」
- 기능 소개 3~4개 (암송·인증·아멘·함께하기) — 짧은 카드 또는 아이콘+텍스트
- 로그인 없이 열람 가능

### 2. 카카오 로그인

- 카카오 브랜드 가이드 준수 (노란 버튼)
- 실명 입력: `TB_MEMBER.NAME` (카카오 닉네임과 별도)
- MVP mock: 로그인 UI만, 실제 OAuth는 추후

### 3. 홈

- 참여 중인 암송방 카드 목록 (`TB_ROOM_MEMBER.STATUS='Y'`)
- 각 카드: 방 이름, 오늘 내 인증 상태 뱃지 (완료/미완료)
- FAB 또는 상단 버튼: 「방 만들기」「초대코드로 입장」

### 4. 암송방 상세

**상단 고정 영역 (접을 수 있음):**
- 현재 암송 말씀 (`TB_RECITATION_SECTION`, `IS_ACTIVE='Y'`)
- (방장 전용) 본문 등록/수정 — `ROOM_ROLE = 'LEADER'`일 때만 표시, 별도 관리자 페이지 없음
- 오늘 내 인증 상태 + 인증 CTA (녹음 / 계수기)

**하단 스크롤 영역:**
- 날짜별 인증 피드 (위 피드 구조, **최신 날짜가 위**, `STATUS='Y'` 카드만)

### 5. 오늘 인증 상태

| 상태 | 표시 |
|------|------|
| row 없음 또는 `STATUS='N'` | 🔴 미완료 — 「오늘 인증하기」CTA |
| `STATUS='Y'` | ✅ 완료 — detail 요약 (선택) |

- 홈 카드·방 상세 **양쪽**에 동일 기준 적용
- `STATUS='N'` 카드는 **인증 피드에 노출하지 않음** — 미완료는 이 영역에서만 표시
- 유효 detail 없이「완료」로 표시하면 안 됨

### 5-1. 방장 본문 등록/수정 (MVP 포함)

- 위치: 암송방 상세 상단, 말씀 표시 영역 안 또는 바로 아래
- 표시 조건: `TB_ROOM_MEMBER.ROOM_ROLE = 'LEADER'`
- 형태: 접을 수 있는 간단한 폼 (제목·범위·본문·진행 여부)
- 별도 관리자 페이지·복잡한 CRUD UI **금지**
- 권한: 프론트 UI 숨김 + 추후 백엔드 재검증

### 6. 녹음 인증

- 녹음 시작/정지 UI
- 미리듣기 → 전송
- 전송 시 `CHECK_IN_TYPE = 'VOICE'`, `AUDIO_URL` 저장 (mock: placeholder URL)
- 본인 detail에 취소(soft delete) 버튼

### 7. 계수기 인증

- 숫자 입력 또는 +/- 버튼
- 전송 시 `CHECK_IN_TYPE = 'COUNTER'`, `COUNTER_VALUE` 저장
- 본인 detail에 취소 버튼

### 8. 아멘 UI

- 위치: **인증 카드 하단**
- 표시: `🙏 아멘 {count}`
- **타인 카드:** 아멘 버튼 활성, 토글 (눌림/해제)
- **본인 카드:** 아멘 버튼 **비활성 또는 숨김** (Service·UI 모두 차단)
- detail 단위 아멘 UI **없음**

## 접근성·PWA

- `manifest.json`: 앱 이름, 아이콘, `display: standalone`, theme-color
- 터치 타겟 최소 44×44px
- 색만으로 상태 구분하지 않음 (아이콘·텍스트 병행)
- 오프라인: MVP에서는 mock data 기준, 추후 Service Worker

---

## MVP 이후 UI (구현하지 않음)

- 통계 대시보드 (개인·방 참여율 차트)
- 서비스 ADMIN 관리 화면
- 푸시 알림 (인증 리마인더)

## 금지 사항

- detail 단위 아멘 UI
- CHECK_IN row 존재 = 완료로 표시
- `STATUS='N'` 카드를 인증 피드에 노출
- PDF에 없는 테이블/컬럼 기반 UI 필드 추가
- 복잡한 관리자 CRUD 테이블 UI — 방장 본문 등록은 방 상세 내 간단 폼만 허용 (MVP)
